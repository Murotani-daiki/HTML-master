タイポグラフィと単位

心地よい「読み体験」を作るためのフォント設定と、柔軟な単位の使い分け

タイポグラフィの重要性：Webサイトの9割は「文字」でできている

Webサイトのデザインにおいて、最も影響力が大きいのは「文字（タイポグラフィ）」です。適切なフォント選びと読みやすい配置は、ユーザーに安心感を与え、情報の理解を助けます。逆に、文字が小さすぎたり行間が詰まっていたりすると、ユーザーはすぐにページを離れてしまいます。

このガイドでは、フォントの指定方法から、現代のWeb制作に不可欠な「相対単位」の使い分けまで、心地よい「読み体験」を作るためのテクニックを詳しく解説します。

CSS：標準的なタイポグラフィ設定 
p {
    font-family: 'Inter', sans-serif; /* フォントの種類 */
    font-size: 1.125rem;            /* 文字の大きさ */
    line-height: 1.8;               /* 行間（読みやすさの鍵） */
    letter-spacing: 0.03em;          /* 字間（少し広げると高級感が出る） */
    color: var(--text-secondary);   /* 文字色 */
}

1. フォントの種類と「フォント・ファミリ」

Webブラウザには、ユーザーの環境（Windows, Mac, iPhoneなど）によってインストールされているフォントが異なります。そのため、CSSでは複数の候補をセットで指定する「フォントスタック」という書き方をします。

  • sans-serif (サンセリフ): ゴシック体に近い装飾のない書体。画面上で読みやすく、現在のWebの主流です。
  • serif (セリフ): 明朝体に近い装飾のある書体。伝統的、あるいは高級感を演出したい時に使います。
  • monospace (等幅): 全ての文字が同じ幅の書体。コードの表示やデータの一覧に向いています。

2. px, rem, em, %：単位の使い分け完全ガイド

「とりあえずpx」で指定してはいませんか？ Webの単位にはそれぞれ役割があります。

px (ピクセル) - 絶対単位

「画面上の1点」を指定する基本単位です。直感的ですが、ブラウザの設定で文字を大きくしたいユーザーの要望を無視してしまうため、文字サイズへの使用は避ける傾向にあります。

rem (レム) - 推奨単位

「ルート（html要素）」のサイズを 1 とした単位です。ルートが 16px なら、2rem は 32px になります。ユーザーがブラウザ設定で文字サイズを変えた際、サイト全体が連動して拡大縮小されるため、アクセシビリティに非常に優れています。

em (エム) - 相対単位

「親要素」のサイズに対する割合です。アイコンと文字のバランスを維持したい時など、コンポーネント内での微調整に向いています。

vw / vh - ビューポート単位

画面の幅や高さに対する割合（100vw = 画面幅100%）です。大きな文字を画面幅に合わせて自動調整したい「フルスクリーンデザイン」で活躍します。

3. 読みやすさを激変させる「余白」の制御

文字サイズと同じくらい重要なのが **`line-height`（行間）** です。日本語の場合、標準設定では行間が狭すぎて読みづらく感じることが多いです。
推奨値は **`1.6` 〜 `1.9`** です。この値は単位をつけずに数値だけ書くのが一般的です（文字サイズに対して自動計算されるため）。

💡 プロのアドバイス：Google Fontsを活用しよう

このサイトでも使われている `Inter` や `Outfit` のように、Googleが無料で提供しているフォントを使うことで、どのデバイスから見ても同じ美しいタイポグラフィを実現できます。ただし、読み込みすぎるとサイトが重くなるので、必要なウェイト（太さ）だけを選びましょう。

よくある質問 (Q&A)

Q: 1rem は何ピクセルですか？

A: 一般的なブラウザの標準設定では **16px** です。したがって、1.5rem = 24px、0.75rem = 12px と計算できます。電卓を叩くのが面倒な場合は、ブラウザの検証ツールで実際のピクセル数を確認できます。

Q: text-align: justify は使ったほうがいいですか？

A: 両端揃えにするプロパティですが、日本語のWebサイトでは不自然に大きな空白が空く（いわゆる「文字の川」現象）ことが多いため、基本的には `text-align: left`（左揃え）が推奨されます。

Q: font-weight の数値指定が効きません。

A: フォント自体がその太さをサポートしている必要があります。例えば 800 (Extra Bold) を指定しても、読み込んでいるフォントに 400 (Regular) と 700 (Bold) しか用意されていなければ、自動的に 700 が代わりに使用されます。

まとめ

タイポグラフィはWebデザインの「声」です。フォント選び、適切な単位の使い分け、そして心地よい行間。これらを一つずつ丁寧に整えることで、あなたのメッセージはより深く、より正確にユーザーへと届くようになります。
