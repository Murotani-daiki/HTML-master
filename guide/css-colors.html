色と背景

サイトの印象を決める色彩設計と、豊かな背景表現をマスターしよう

色の魔法：ユーザーの感情をデザインする

Webサイトを訪れたとき、最初に私たちの心に飛び込んでくるのは「色」です。明るい青は信頼感を、情熱的な赤は力強さを、深緑は安らぎを与えます。CSSでの色彩設計は、単なる装飾ではなく「ユーザー体験（UX）」そのものを左右する重要な要素です。

このガイドでは、基本の色指定から、アクセシビリティ（見やすさ）を考慮した色の選び方、そしてモダンな背景表現まで、プロが現場で使うテクニックを網羅的に解説します。

1. CSSで使える4つの色形式

目的や好みに合わせて、以下の4つの形式を使い分けましょう。

  • HEX値（16進数）: `#38bdf8` のように書きます。最も一般的で、グラフィックツールからのコピー＆ペーストにも適しています。
  • RGB / RGBA: `rgb(56, 189, 248)` と指定します。`rgba` を使うと4つ目の数値で透明度（0.0〜1.0）をコントロールでき、重ね合わせの表現に最適です。
  • HSL: `hsl(200, 100%, 50%)` のように、色相・彩度・輝度で指定します。「もう少し明るくしたい」という微調整が数値だけで行えるため、開発者に人気があります。
  • OKLCH (モダン): 最新のCSSで使用可能。人間の目にとって「均一な明るさ」を保ったまま色を調整できる、次世代の指定形式です。

2. 背景（background）を制する

単色の塗りつぶし以外にも、CSSは豊かな背景表現を可能にします。

グラデーションの表現

現在のWebデザインで欠かせないのが `linear-gradient` です。方向、色の変化、色の割合を自由に操ることができます。

CSS：グラデーション 
/* 右下に向かって変化するグラデーション */
background: linear-gradient(135deg, #38bdf8, #818cf8);

背景画像の高度な制御

背景画像を表示するとき、ただ読み込むだけでは不十分です。以下のプロパティを組み合わせて「画面にフィット」させましょう。

background-image: url("hero.webp");
background-size: cover;      /* 隙間なく画面を埋める */
background-attachment: fixed; /* スクロールしても背景が動かない効果 */
background-position: center;  /* 常に中央を表示 */

3. 重要なルール：色のアクセシビリティ

どんなに美しい色でも、文字が読めなければWebサイトとしての価値を失います。色のコントラスト（明暗の差）には注意を払いましょう。

⚠️ コントラスト比の基準 (WCAG)

通常のテキストの場合、背景とのコントラスト比は少なくとも **4.5:1** 以上であることが推奨されています。ライトグレーの背景に白文字のような組み合わせは避けましょう。Google Chromeの検証ツールでもこの比率をチェックできます。

4. CSS変数（カスタムプロパティ）による色の管理

大規模なサイトでは、色を一箇所で管理するのが賢い方法です。CSS変数を使えば、ブランドカラーの変更も一瞬で完了します。

:root {
    --primary-color: #38bdf8;
}

h1 {
    color: var(--primary-color);
}

よくある質問 (Q&A)

Q: 透過させるには opacity と RGBA どちらが良いですか？

A: `opacity` はその中の文字まで透けてしまいます。**「背景だけ透かしたい」場合は、`rgba` や `hsla` を使うのが正解です。**

Q: 3桁のHEX値（例: #fff）は何ですか？

A: 短縮記法です。`#ffffff` は `#fff`、`#aabbcc` は `#abc` と省略して書くことができます。ファイルサイズを節約するのに役立ちます。

Q: ダークモードに対応するには？

A: メディアクエリ `@media (prefers-color-scheme: dark)` を使い、変数に代入した色を上書きするのがモダンな手法です。

まとめ

色の指定は単なるコーディング作業ではなく、ユーザーへのメッセージです。色の形式、アクセシビリティ、そしてモダンな管理手法をマスターして、視覚的に心地よく、かつ機能的なウェブサイトを目指しましょう！
