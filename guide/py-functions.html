関数

レシピをまとめて名付ける：処理の部品化でコードをスッキリさせよう

関数とは？：処理をパッケージ化する「魔法のレシピ」

プログラミングにおいて、何度も使う一連の処理をひとまとめにして名前をつけたものを「関数（Function）」と呼びます。

料理に例えるなら、関数は「特定の料理を作るためのレシピ」のようなものです。一度レシピを書いておけば、材料（データ）を渡すだけで、何度でも同じ味（結果）を再現できます。大規模な開発では、プログラムを一から長く書くのではなく、小さくて便利な「関数の部品」を組み合わせて巨大なシステムを構築していきます。

1. 関数の作り方：def キーワード

Pythonでは `def`（Defineの略）キーワードを使って関数を定義します。関数名には、変数と同じく **スネークケース (snake_case)** を使うのが一般的です。

Python：関数の定義 
def celebrate():
    print("おめでとうございます！")
    print("関数の実行に成功しました。")

# 呼び出し（実行）
celebrate()

コロン `:` の後のインデントされた範囲が、その関数の一部として扱われます。インデントが終わった場所が、関数の終わりの合図です。

2. 引数：関数にデータを渡す

関数は外からのデータ（引数）を受け取ることで、より柔軟に動くようになります。

デフォルト引数

あらかじめ値を決めておくことで、引数が渡されなかった時の動作を指定できます。

Python：引数の活用 
def greet(name, message="こんにちは"):
    print(f"{name}さん、{message}！")

greet("田中")          # 田中さん、こんにちは！
greet("佐藤", "お疲れ様") # 佐藤さん、お疲れ様！

3. 戻り値：処理結果を外に返す

関数の中で計算した結果を、呼び出し元に返したい時に `return` を使います。戻り値がない関数は、Python内部ではこっそり `None` という値を返しています。

Python：戻り値 
def add_numbers(a, b):
    return a + b

total = add_numbers(10, 5) # totalに15が代入される

複数の値を返す

Pythonの関数は、カンマで区切るだけで複数の結果を一度に返すことができます（実体はタプルという形式です）。

Python：複数戻り値 
def get_min_max(numbers):
    return min(numbers), max(numbers)

low, high = get_min_max([1, 5, 10, 2])
# low=1, high=10 となる

4. スコープ：変数の「有効範囲」を知る

関数の中で作った変数は、その関数の外側から使うことはできません。これを **ローカルスコープ** と呼びます。逆に、プログラムのトップレベルで作った変数はどこからでも見えますが（グローバルスコープ）、関数の中から書き換えるには `global` 宣言が必要になります。

🚀 プロの習慣：Docstring（ドキュメンテーション）

関数のすぐ下に `""" 文字列 """` を書くことで、その関数が「何をするものか」を説明できます。これを **Docstring** と呼び、大規模なコードを読む際の大きな助けになります。

def square(x):
    """渡された数値を2乗して返します。"""
    return x * x

よくある質問 (Q&A)

Q: 関数の中で関数を呼び出すことはできますか？

A: もちろん可能です！ 大きな問題を小さな機能（関数）に分解し、それらを組み合わせて解決するのがプログラミングの王道です。

Q: lambda（ラムダ）式とは何ですか？

A: 名前をつけない短い関数のことで「無名関数」とも呼ばれます。一行ですむような単純な処理を引数として渡したい時によく使われます。

Q: 同じ名前の関数を2つ作るとどうなりますか？

A: 後から書いた方の関数で上書きされてしまいます。Pythonには「オーバーロード（引数によって同名の関数を使い分ける機能）」はないので注意しましょう。

まとめ

関数は、あなたのコードを「ただの命令」から「再利用可能な知恵のパーツ」へと変えてくれます。一つひとつの関数をシンプルに、分かりやすい名前で保つことが、美しいプログラムを書くための最大の秘訣です。
