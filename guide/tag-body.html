<body>

本文（body）

ユーザーが目にする文字、画像、ボタン、動画など、すべてのコンテンツがここに含まれます。

bodyタグとは？：ユーザーが見る全ての「世界」の土台

Webブラウザがひとつの「舞台」だとすれば、その舞台上で繰り広げられる全てのドラマ（コンテンツ）を保持するのが、この `<body>` タグです。先ほど学んだ `<head>` タグが「脳」であり、目に見えない思考や設定を司るのに対し、 `<body>` は文字通り「体」であり、ユーザーが実際に目で見て、触れて、心に刻む情報の全てを詰め込む箱となります。

このタグの開始と終了の間に書かれたものだけが、ブラウザのキャンバス上に描画されます。見出し、段落、画像、リンク、そしてこのガイドを読んでいるあなたのブラウザに表示されている全ての装飾も、元を辿ればこのbodyタグという大地の上に成り立っています。

bodyの標準的な構成

<body>
  <header>サイトのヘッダー</header>
  <main>メインコンテンツ</main>
  <footer>フッター情報</footer>
</body>

1. 画面表示の司令塔としての役割

bodyタグは、ページ全体のスタイリングを決定する際にも重要な役割を果たします。例えば、サイト全体の背景色や文字の色、フォントの基本サイズをCSSで指定する時、そのターゲットとなるのは多くの場合この `<body>` 要素です。

また、DOM（Document Object Model）という仕組みにおいては、画面に見える全ての要素の「唯一の親」となります。JavaScriptでページの内容を動的に変える時も、このbodyを起点として様々な要素へアクセスしていくことになります。

2. グローバル属性とカスタマイズ

bodyタグには、サイト全体を制御するための便利な属性を付けることができます。

  • id / class: ページごとに異なるクラスを付けることで、「トップページだけ特別な背景にする」といった条件分岐が容易になります。
  • data-* 属性: JavaScriptで使用するためのカスタムデータを埋め込むことができます。例えば、 `data-theme="dark"` としておくことで、スクリプトが読み込まれた瞬間にダークモードの設定を反映させるといった高度な制御が可能です。

3. ブラウザごとの「癖」に注意

実は、何もCSSを書いていない状態でも、bodyタグにはブラウザごとに微妙な「余白（margin）」が設定されています。多くのモダンブラウザでは **8px** 程度の余白が自動でつきます。「画面の端にぴったり画像がつかない」という現象が起きた時は、CSSで `body { margin: 0; }` と指定することで解決できます。

よくある質問 (Q&A)

Q: 1つのHTMLに2つ以上の body タグを書いてもいいですか？

A: **絶対にダメです。** 文法上、bodyタグは1つのHTML文書に **たった1つだけ** 存在しなければなりません。複数書くとブラウザが混乱し、表示が崩れたりJavaScriptが動かなくなったりする原因になります。

Q: bodyタグを省略しても表示されるのはなぜ？

A: htmlやheadタグと同様、ブラウザには「不足しているタグを推測して補う」という超人的な機能があるからです。しかし、これは「ブラウザの温情」に甘えている状態であり、SEOやアクセシビリティの観点からは非常に好ましくありません。正しく記述するのがWeb制作者の誠実さです。

Q: bodyタグ自体に背景画像を設定できますか？

A: はい、非常に一般的です。CSSで `body { background-image: url(...); }` と指定することで、サイト全体に広がる背景画像を簡単に配置できます。その際は、文字が見づらくならないよう色のコントラストに注意しましょう。

まとめ

bodyタグは、あなたのメッセージを形にするための「広大なキャンバス」です。ここをどう彩り、どう整理するか。その一つひとつの選択が、ユーザーにとっての「Web体験」そのものになります。HTML Masterで学ぶ全てのタグは、このbodyという最高の舞台で輝くために存在しています。
