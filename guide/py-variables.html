変数と代入

データに名前（ラベル）をつけて、再利用できるようにしよう

変数とは？データの「名札」

プログラミングにおいて、変数（Variable）は最も基本的かつ重要な概念です。多くの解説書では「データを入れる箱」と例えられますが、Pythonにおいては「データにつける名札（ラベル）」と考える方がより正確です。

例えば、広大なメモリ空間のどこかに「100」という数値データが存在しているとします。このデータに `score` という変数を割り当てることは、「100」というデータに `score` というタグ紐を付けるようなものです。以降、プログラム中で `score` と書けば、Pythonは即座にそのタグ紐をたぐり寄せ、結びついている「100」という値を参照します。

変数の定義と代入

Pythonでの変数の定義は非常にシンプルです。型宣言（「これは整数です」といった前置き）は不要で、いきなり名前を書いて値を代入（Assign）するだけです。

Python 
player_name = "勇者"  # 文字列を代入
level = 1            # 数値を代入
is_active = True     # 真偽値（True/False）を代入

print(f"{player_name}はレベル{level}です")

ここで使われている `=` は、数学の「等しい」という意味ではありません。「右側の値を、左側の変数名に結びつける」という命令です。

Pythonの命名規則：PEP 8

Pythonには「PEP 8（ペップエイト）」という公式のスタイルガイドがあり、世界中のPythonエンジニアがこれを守ってコードを書いています。変数名に関しては以下のルールが推奨されています。

✅ 推奨される書き方：スネークケース

全て小文字で書き、単語の区切りをアンダースコア `_` で繋ぎます。

  • `user_name`
  • `total_price`
  • `max_hit_point`

❌ 避けるべき書き方

  • `userName` (キャメルケース)：JavaやJavaScriptで一般的ですが、Pythonでは使いません。
  • `User_Name` (パスカルケース)：Pythonでは「クラス名」に使われるため、変数には使いません。
  • 意味のない名前：`a` や `data` だけでは、後で読み返した時に何の値か分からなくなります。

定数について

Pythonには、値を変更できない「定数」専用の構文が存在しません。しかし、慣習として「全て大文字」で書くことで、「これは定数だから書き換えないでね」という意思表示をします。

TAX_RATE = 1.1       # 消費税率（定数扱い）
MAX_USERS = 100      # 最大ユーザー数

動的型付けと型ヒント

Pythonは「動的型付け言語」です。これは、変数の中身が数値であれ文字列であれ、Pythonが実行時に自動的に判断してくれることを意味します。柔軟にコードが書ける反面、大規模な開発では「この変数には何が入るんだっけ？」と混乱する原因にもなります。

そこで、現代のPython（バージョン3.5以降）では、「型ヒント（Type Hinting）」という機能が使われるようになりました。

Python (型ヒント) 
# : str や : int を付けて、何が入るべきかを明示する
user_name: str = "Taro"
age: int = 20

# これでもエラーにはならないが、エディタが警告を出してくれる
age = "二十歳"

型ヒントを使うことで、コードの意図が明確になり、バグを未然に防ぐことができます。

変数の有効範囲（スコープ）

変数は、定義された場所によって使える範囲が決まっています。関数の中で定義した変数は「ローカル変数」と呼ばれ、その関数の外からは見えません。逆に関数の外で定義された変数は「グローバル変数」と呼ばれ、どこからでも参照できます（ただし、関数内での書き換えには注意が必要です）。

まとめ

たかが変数、されど変数です。わかりやすく適切な名前が付けられた変数は、それだけで「何をするコードなのか」を雄弁に語ります。美しいコードを書く第一歩は、美しい変数を定義することから始まります。
