変数と代入

データに名前（ラベル）をつけて、再利用できるようにしよう

変数とは？：データに名前をつける「ラベル」

プログラミングにおける **変数** とは、数値や文字列などのデータに名前をつけて保存しておく仕組みのことです。よく「箱」に例えられますが、Pythonにおいては「データという実体に名前ラベルを貼り付ける」というイメージの方が正確です。

変数を使うことで、同じデータを何度も入力する手間が省けるだけでなく、プログラムの意図が読み手に伝わりやすくなり、メンテナンス（修正）も格段に容易になります。

1. 基本の書き方と代入

Pythonでは、変数名、イコール `=` 、値をこの順で記述します。これを「代入（Assignment）」と呼びます。

Python：基本 
name = "Python Master"
version = 3.12
is_published = True

JavaScriptのように `const` や `let` といった宣言キーワードは必要ありません。名前を書いた瞬間に変数が作られます。

2. Pythonの大きな特徴：動的型付け

Pythonは **動的型付け (Dynamic Typing)** 言語です。これは、変数が作成される時に「この変数は整数用」「これは文字列用」とあらかじめ決める必要がないことを意味します。中に入れるデータによって、Pythonが実行時に最適な「型」を自動的に判断してくれます。

Python：柔軟な型変更 
x = 100    # 最初は整数の型 (int)
x = "Hello" # 後から文字列 (str) を入れてもエラーにならない

この柔軟性は素早い開発を可能にしますが、意図しない型が混じってしまうバグの原因にもなるため、注意深い管理が必要です。

3. 命名規則：スネークケースの徹底

Pythonには公式のスタイルガイド **PEP 8** があり、変数の付け方にも明確なルール（慣習）があります。

✅ 推奨：snake_case (スネークケース)

単語を小文字で書き、アンダースコア `_` で繋ぎます。 例: `user_name`, `total_price`, `max_level`

❌ 非推奨：camelCase / PascalCase

Pythonの世界では、クラス名以外で大文字を混ぜる書き方はあまり一般的ではありません。 例: `userName` (避けるべき)

4. 高度なテクニック：同時代入と定数

Pythonには、コードを簡潔にするためのユニークな代入方法がいくつかあります。

複数代入（アンパック）

一行で複数の変数に値を割り当てることができます。

Python：アンパック 
x, y, z = 10, 20, 30
# xは10, yは20, zは30になる

Pythonにおける「定数」

実は、Pythonには言語レベルで「絶対に値を変更できない定数」は存在しません。しかし、プログラマ同士の約束事として **「すべて大文字で書かれた変数は定数として扱い、後から書き換えない」** というルールがあります。

Python：定数の慣習 
API_KEY = "secret_key_123" # 大文字は書き換えない約束

5. モダンPythonの新常識：型ヒント

近年では、大規模なプログラムでもバグを防ぎやすくするために、変数に「期待する型」を添える **型ヒント (Type Hints)** が広く使われています。

Python：型ヒント活用 
score: int = 100
user_name: str = "Alice"

よくある質問 (Q&A)

Q: 日本語の変数名は使えますか？

A: 技術的には使えますが（例: `名前 = "田中"`）、現場では **100% 英語（アルファベット）** が使われます。プログラミングの世界標準に合わせるためにも、英語での命名を強くお勧めします。

Q: 予約語（キーワード）とは何ですか？

A: `if`, `for`, `def` など、Pythonが特別な意味として使っている単語のことです。これらは変数名として使うことはできません。

Q: 変数を削除することはできますか？

A: `del variable_name` を使うと削除できます。ただ、通常の開発で意図的に削除が必要になる場面は非常に稀です。

まとめ

変数は、コンピュータに指示を出すための「言葉」の一部です。適切な名前をつけ、適切な型を意識することで、あなたのコードは魔法のような読みやすさと安定感を手に入れます。最初はスネークケースに慣れるところから始めてみましょう！
