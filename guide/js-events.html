イベント

ボタンが押されたら？：ユーザーの「きっかけ」を機能に変える

イベント駆動型プログラミングの基礎

JavaScriptの最大の特徴は、ユーザーの操作やブラウザの状態変化（＝イベント）をトリガーにしてプログラムを動かせることです。「クリックしたらメニューが開く」「スクロールしたら画像がふわっと現れる」といった動作は、すべてイベントによって制御されています。

主要なイベントの種類

ブラウザで発生するイベントは多岐にわたりますが、頻出するものをグルーピングしてみました。

マウス・タッチ操作

click, dblclick (ダブルクリック), mouseover (乗せた時), touchstart (タップ開始)

キーボード操作

keydown (押した瞬間), keyup (離した瞬間), change (入力確定)

フォーム操作

submit (送信), focus (選択中), blur (選択解除)

ウィンドウ・読み込み

load (完了時), scroll (スクロール時), resize (画面サイズ変更)

イベントリスナー：addEventLister

HTMLの中に直接 `onclick="..."` と書くのは古いやり方です。現代では `addEventListener` メソッドを使って、HTMLとJavaScriptをきれいに分離するのが定石です。

JavaScript 
const button = document.querySelector(".btn");

// basic syntax: 要素.addEventListener("イベント名", 関数)
button.addEventListener("click", () => {
    console.log("ボタンが押されました！");
    button.classList.add("active");
});

イベントオブジェクト (Event Object)

イベントが発生したとき、その詳細情報（座標、押されたキー、ターゲット要素など）が詰まった「イベントオブジェクト」が関数に渡されます。

document.addEventListener("keydown", (e) => {
    // e は event の略
    if (e.key === "Enter") {
        console.log("エンターキーが押されました");
    }
});

デフォルト動作のキャンセル

フォームの送信やリンクの遷移など、ブラウザが標準で持っている動作を止めたい場合は `preventDefault()` を使います。

form.addEventListener("submit", (e) => {
    e.preventDefault(); // ページリロードを防ぐ
    console.log("JavaScriptで送信処理を行います");
});

まとめ

イベントを制する者がインタラクティブWebを制します。まずは `click` イベントから始めて、ユーザーのあらゆるアクションに反応する楽しいサイトを作ってみてください。
