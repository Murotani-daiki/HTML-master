ボタンが押されたら？：ユーザーの「きっかけ」を機能に変える

Webを「動かす」きっかけ：イベントの本質

Webサイトは、ただそこにあるだけの「静的な文書」から、ユーザーの動きに反応する「動的なアプリケーション」へと進化しました。この進化の主役が **イベント（Event）** です。

「ボタンをクリックする」「画像をマウスでなぞる」「スマホで画面をスワイプする」「入力フォームを送信する」など、ブラウザ上で発生するあらゆるアクションを、JavaScriptは「イベント」という信号として受け取ります。この信号をキャッチし、特定のプログラムを実行させることで、私たちは初めて「生きているWebサイト」を作ることができるのです。

1. イベントを検知する：addEventListener

モダンなJavaScript開発において、イベントを待機させるための標準的な方法が `addEventListener()` メソッドです。これは特定のHTML要素に対して「特定のイベントが発生するのを、耳を澄ませて待っていなさい」という命令を下すものです。

JavaScript：基本の書き方 
const btn = document.querySelector("#submit-btn");

// 第一引数：イベントの種類、第二引数：実行する関数（コールバック）
btn.addEventListener("click", () => {
    console.log("ボタンが元気にクリックされました！");
});

2. 主要なイベントとそのカテゴリ

ブラウザが検知できるイベントは数百種類に及びますが、まずは以下の主要なものを抑えましょう。

マウス系

click, dblclick, mouseenter, mouseleave

キーボード系

keydown, keyup, keypress

フォーム系

submit, input, change, focus, blur

ページ系

DOMContentLoaded, load, scroll, resize

3. イベントオブジェクト：詳細情報の宝庫

イベントが発生した際、ブラウザは自動的に「その時、何が起きたか」を記した **イベントオブジェクト (event)** を関数に渡してくれます。「どのボタンが押されたか」「画面のどの座標か」「どのキーが叩かれたか」を知るには、これを使います。

JavaScript：情報の活用 
document.addEventListener("keydown", (event) => {
    console.log(`押されたキーのコード: ${event.code}`);
    if (event.key === "Enter") {
        console.log("Enterキーが押されました！");
    }
});

4. イベントの伝搬：バブリングとキャプチャリング

JavaScriptのイベントには「泡（バブル）」のように上へ広がる性質があります。例えば、ボタンをクリックすると、そのボタンだけでなく、その親要素（div）、さらにその親（body）へとクリックイベントが連鎖的に伝わっていきます。これを **バブリング (Bubbling)** と呼びます。

💡 event.stopPropagation() の使いどころ

子要素のクリックだけで処理を終わらせ、親要素のイベントを発火させたくない場合は `event.stopPropagation()` を呼び出します。これにより、イベントの連鎖を断ち切ることができます。

5. 標準動作をキャンセルする：preventDefault()

ブラウザには「リンクをクリックすると別ページへ飛ぶ」「フォームを送信するとページが更新される」といった、最初から決まっている動き（既定の動作）があります。JavaScriptでこれを制御したい場合に使うのが `event.preventDefault()` です。

JavaScript：送信キャンセル 
const form = document.querySelector("#my-form");
form.addEventListener("submit", (e) => {
    e.preventDefault(); // ページの再読み込みを防ぐ
    console.log("送信を一時停止し、JSでチェックを行います。");
});

6. 効率的な戦術：イベントデリゲーション

例えば100個のリスト項目一つ一つに `addEventListener` を設定するのは、メモリを消費し、コードも複雑になります。代わりに、それらを包む親要素に一つだけ設定し、バブリングしてきたイベントを親がまとめて処理する手法を **イベントデリゲーション（委譲）** と呼びます。これは大規模開発において必須のテクニックです。

よくある質問 (Q&A)

Q: onclick プロパティと addEventListener、どちらが良いですか？

A: 断然 **addEventListener** です。一つの要素に複数の異なる処理を登録でき、あとから解除するのも簡単だからです。

Q: スマホの「タップ」は click で動きますか？

A: はい、動きます。ただし、より高速に反応させたり、複数本の指による操作を扱いたい場合は `touchstart` などのタッチイベントを使用します。

Q: addEventListener を解除する必要はありますか？

A: 画面から消える要素に重い処理を仕掛けている場合は、`removeEventListener` で解除しないと、ブラウザのメモリが徐々に減っていく「メモリリーク」の原因になることがあります。

まとめ

イベントを使いこなせば、ただの表示物だったWebページが、ユーザーと対話する「アプリケーション」へと進化します。バブリングや既定の動作の制御など、奥が深い世界ですが、これらを自由に操れるようになれば、あなたのJavaScriptの実力は一段上のレベルに到達します。まずは一番身近な `click` イベントから、世界を動かしていきましょう！
