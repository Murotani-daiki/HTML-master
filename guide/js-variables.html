変数と定数

let と const：データを賢く管理するための第一歩

データを「保存」する：変数の本質

プログラミングにおいて、データ（数値や文字など）を一時的にメモリ上に保存しておく場所を「変数」と呼びます。保存したデータには名前をつけることができ、その名前を使って後からデータを呼び出したり、計算に使ったりすることができます。

現代のWebアプリケーションでは、単に値を保持するだけでなく、アプリケーションの状態（State）を管理するために変数が極めて重要な役割を果たします。例えば、ユーザーのログイン状態、ゲームのスコア、アニメーションの進行具合など、動的に変化する情報のすべてが変数として管理されています。

💡 変数は「ラベル付きの箱」

データという「中身」を箱に入れ、分かりやすい「ラベル（名前）」を貼るイメージで捉えると理解しやすくなります。良いプログラムを書く第一歩は、このラベルに「中身が何であるか」を明確に示す名前をつけることです。

キーワードの進化：const, let, そして var

JavaScriptには変数・定数を宣言するための3つのキーワードがあります。言語の進化とともに、より安全で予測可能なコードを書くための仕組みとして整理されてきました。

1. const (定数)：デフォルトで使うべき選択肢

一度代入（初期化）したら、後から値を変更（再代入）できない「再代入不可の箱」です。

意図しない値の書き換えを防ぐことができるため、「まずはすべて const で書き始め、どうしても再代入が必要な場合だけ let に変える」というのが現代のプロフェッショナルな開発の鉄則です。これにより、プログラムの挙動が安定し、バグの混入を大幅に減らすことができます。

JavaScript 
const API_ENDPOINT = "https://api.example.com";
const MAX_RETRY = 3;

// API_ENDPOINT = "http://other.com"; // エラー：再代入はできません

2. let (変数)：変化を許容する

値を後から自由に書き換えることができる「再代入可能な箱」です。ループ処理のカウントアップ（i++）や、検索条件の変更、計算の途中経過を保持する場合などに使用します。

JavaScript 
let currentScore = 0;
currentScore += 100; // 100点を加算
currentScore = 250;  // 最終的な計算結果で上書き

⚠️ 過去の遺物：var

ES2015（ES6）以前のJavaScriptでは `var` が唯一の手段でしたが、現在では「巻き上げ（Hoisting）」による予期せぬ挙動や、スコープの曖昧さが原因でバグを誘発しやすいため、新しいコードで使うことはまずありません。

メモリとスコープの仕組み

変数が「どこで有効か」を決定するのが **スコープ（Scope）** です。`let` と `const` は「ブロックスコープ」を採用しています。

これは、波括弧 `{ }` で囲まれた範囲内だけで変数が生きるというルールです。これにより、同じ変数名を使っても場所が違えば互いに干渉しなくなり、大規模なプログラムでも安心して開発ができるようになっています。

JavaScript：スコープの例 
if (true) {
    const message = "ブロック内";
    console.log(message); // 表示される
}
// console.log(message); // エラー！ブロックの外からは見えません

命名規則：読みやすいコードのための作法

変数名は自由につけることができますが、チーム開発や未来の自分のために、JavaScriptコミュニティの標準的なルールに従いましょう。

  • キャメルケース (camelCase): `userName` や `totalPrice` のように、2つ目以降の単語の頭文字を大文字にします。
  • 名詞であること: 変数は「物」なので、`getUser`（動詞：関数向き）ではなく `userList`（名詞）のような名前にします。
  • ブール値の命名: 真偽値（true/false）を入れる場合は、`isActive` や `hasPermission` のように、疑問文のような名前にすると読みやすくなります。

よくある質問 (Q&A)

Q: 全く使い分けが分かりません。どっちを使えばいいですか？

A: 迷ったら **const** です。プログラムのほとんどの値は、初期化後に変更する必要がありません。変更しようとしてエラーが出たら、その時だけ **let** に書き換える、というスタイルが最も安全です。

Q: const で宣言した配列やオブジェクトの中身は変えられますか？

A: はい、変えられます。`const` が禁止しているのは「箱自体の入れ替え（再代入）」であり、箱の中身（配列の要素やオブジェクトのプロパティ）を操作することは可能です。これが「const は定数ではなく再代入不可（Immutable Binding）である」と言われる理由です。

Q: 変数の宣言を忘れて代入したらどうなりますか？

A: エラー（ReferenceError）が発生するか、意図しないグローバル変数が作成されてしまう危険があります。必ず `const` か `let` をつけて宣言することを習慣にしましょう。

まとめ：良い変数管理とは

変数は単なるデータの置き場ではなく、プログラムの意図を他人に伝えるための「ドキュメント」でもあります。適切なキーワード（主に const）を選び、適切なスコープで、適切な名前をつける。この基本的な積み重ねが、バグのない美しいアプリケーションを生み出す力（パワー）となります。
