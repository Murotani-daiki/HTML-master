ループ処理

コンピュータの得意技：めんどくさい繰り返しをプログラムで解決

ループ処理とは？プログラミングの「加速装置」

コンピュータが人間より圧倒的に優れている点、それは「文句を言わずに、同じ作業を、超高速で何万回でも繰り返せる」ことです。この能力を引き出すのがループ処理です。「1万件の顧客データから東京都の人だけを抜き出す」「ゲームのキャラクター100体を一度に動かす」といった処理は、ループを使えばたった数行のコードで実現できます。

基本にして王道：for 文

回数が決まっている繰り返しの定番です。「開始値」「終了条件」「増え方」の3つをセットで指定します。

JavaScript 
for (let i = 1; i <= 5; i++) {
    console.log(`${i}回目の周回です`);
}

このコードは、変数 `i` が1からスタートし、5以下の間はずっと繰り返され、1周するごとに `i` が1ずつ増える、という意味になります。

配列との相性抜群：for...of 文

モダンなJavaScriptでは、配列の中身を順番に取り出すために `for...of` という便利な構文が用意されています。従来の `for` 文よりもミスが起きにくく、コードも直感的です。

JavaScript 
const colors = ["Red", "Green", "Blue"];

// colorsから1つずつ取り出して color に入れる
for (const color of colors) {
    console.log(`色は ${color} です`);
}

条件が満たされるまで：while 文

回数が決まっていなくても、「ある条件がクリアされるまで続ける」場合に適しています。例えば「RPGでHPが0になるまで戦う」といったシチュエーションです。

let hp = 10;
while (hp > 0) {
    console.log(`勇者の攻撃！ HP残り: ${hp}`);
    hp = hp - 3; // これを忘れると無限ループ！
}
console.log("モンスターを倒した！");

⚠️ 恐怖の無限ループ

終了条件が決して満たされないループを作ってしまうと、プログラムは永遠に走り続け、ブラウザがフリーズ（応答なし）してしまいます。`while` 文を使うときは、「必ずいつか条件がfalseになるか」を慎重に確認しましょう。

ループの制御：break と continue

ループの途中で流れを変えたい場合に使います。

  • break: ループを強制的に「脱出」します。「探し物が見つかったらそこで終了」といった時に使います。
  • continue: 今の周回をスキップして、「次の周回」に進みます。「特定の値だけ無視したい」時に便利です。

まとめ

ループ処理をマスターすることで、あなたは「単純作業」から開放され、よりクリエイティブなロジック作りに集中できるようになります。まずは `for...of` で配列を操作するところから慣れていきましょう。
