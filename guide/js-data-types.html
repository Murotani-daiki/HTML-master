データ型

JavaScriptが扱う情報の「種類」をマスターしよう

データには「型」がある：JavaScriptの分類学

プログラミングで扱うあらゆるデータには、それぞれ決まった種類（型）があります。JavaScriptは、メモリ上のデータを数値として計算し、文字として画面に表示し、あるいは複雑な情報の集合体として管理するために、この「データ型」を内部で厳密に区別しています。

JavaScriptのデータ型は、大きく分けて **「プリミティブ型」** と **「オブジェクト型」** の2つのグループに分類されます。この違いを理解することが、思わぬバグを防ぎ、効率的なコードを書くための最大の鍵となります。

String (文字列)

名前や文章など、テキストを扱うプリミティブ型です。不変（Immutable）という特徴があります。

Number (数値)

整数や小数。JavaScriptでは全て「64ビット浮動小数点数」として扱われます。

Boolean (真偽値)

`true` か `false`。条件分岐（if文）の心臓部となる、最もシンプルな型です。

Special (特殊値)

`null` や `undefined`。値が存在しない、あるいは準備中の状態を表します。

1. プリミティブ型：不変の基礎データ

数値や文字列などのプリミティブ型は、一度作成されたらその値自体を変更することはできません（不変性）。例えば、文字列の特定の一文字だけを書き換えることはできず、変更したい場合は新しい文字列を再作成する必要があります。

文字列 (String) の進化：テンプレートリテラル

引用符 `"` や `'` 以外に、バックティック ( ` ) を使うことで、改行を含んだり、`${ }` を使って変数を直接埋め込んだりすることができます。これは現代のJavaScript開発において標準的な書き方です。

JavaScript：文字の埋め込み 
const lang = "JS";
const message = `私は ${lang} Master を目指しています。
毎日学習しています！`;
console.log(message);

2. 特殊な値：null と undefined の違い

どちらも「値がない」ことを意味しますが、使われる文脈が明確に異なります。

  • undefined: 「まだ値が決まっていない」状態。定義しただけの変数などはこれになります。
  • null: 「意図的に空である」と明示した状態。開発者が明示的に代入するものです。

💡 歴史的なバグ：typeof null

`typeof null` を実行すると `"object"` が返ってきます。これはJavaScriptの最初期からある有名なバグですが、互換性のために修正されずに残っています。「null はオブジェクトではないが、typeofの結果はobjectになる」と覚えておきましょう。

3. オブジェクト型：参照による管理

配列（Array）やオブジェクト（Object）は、複数のデータをまとめて管理するための複雑な仕組みです。プリミティブとは異なり、これらは **「参照（リファレンス）」** で扱われます。

変数にはデータそのものではなく、「データがメモリ上のどこにあるかという住所（参照）」が保存されます。そのため、同じオブジェクトを複数の変数で共有することができ、一つの変数を経由して中身を書き換えると、他の変数から見た中身も変わる、という挙動をします。

JavaScript：参照の仕組み 
const a = [1, 2, 3];
const b = a; // 参照（住所）をコピー
b.push(4);    // bを操作したつもりでも...
console.log(a); // [1, 2, 3, 4] になっています！

4. 動的な型変換（型強制）の落とし穴

JavaScriptは、異なる型同士の計算をしようとすると、良かれと思って自動的に型を変換してしまいます（暗黙の型変換）。これがしばしば混乱の原因となります。

JavaScript：型の衝突 
console.log(1 + "1");  // "11" （数値が文字列に変換されて連結される）
console.log("5" - 2);  // 3    （文字列が数値に変換されて計算される）
console.log("abc" * 2); // NaN  （数値に変換できず "Not a Number" となる）

よくある質問 (Q&A)

Q: 配列 (Array) はオブジェクトの一種ですか？

A: はい、その通りです。JavaScriptの内部では、配列は「数字のキーを持つ特殊なオブジェクト」として実装されています。そのため、`typeof []` は `"object"` になります。

Q: 非常に大きな数値を扱うにはどうすればいいですか？

A: `Number` 型では扱いきれない巨大な整数（2の53乗以上）のために、**BigInt** 型が用意されています。数値の末尾に `n` をつける（例：`100n`）ことで使用できます。

Q: 型を厳密にチェックするには？

A: 比較には必ず `===`（厳密等価演算子）を使いましょう。`==` だと `1 == "1"` が true になってしまいますが、`===` ならデータ型までチェックするため安心です。

まとめ

データ型を正しく理解することは、意図しない計算ミスを防ぎ、大規模で複雑なプログラムを構築するための土台となります。一見ややこしい「参照」や「型変換」のルールも、仕組みを知れば強力な味方になります。今、自分の書いたコードの変数が「どの型」なのか、常に意識することから始めましょう。
