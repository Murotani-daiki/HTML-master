条件分岐

プログラムに「判断」をさせ、処理の流れを切り替えよう

条件分岐とは？：プログラムに「思考」を持たせる

私たちの日常生活は判断の連続です。「信号が赤なら止まる」「雨が降っているなら傘を持つ」といった論理的な判断を、プログラムの世界で実現するのが **条件分岐 (Conditional Branching)** です。

適切な条件分岐を書けるようになると、プログラムは単なる命令の羅列から、状況を自ら判断して最適に動く「知的なシステム」へと進化します。ユーザーの入力チェック、ログイン判定、複雑な計算アルゴリズムなど、あらゆる動的処理の核心となる技術です。

1. Pythonの象徴：インデントによるブロック構造

Pythonが他の言語（JavaScriptやC++など）と決定的に違うのが、波カッコ `{ }` を使わずに「字下げ（インデント）」でコードの範囲を決める点です。

Python：基本の if 
age = 18
if age >= 18:
    print("あなたは成人です")
    print("選挙権があります") # インデントが同じなら同じブロック

条件式の末尾には必ずコロン `:` をつけます。その後に続くインデントされた行が、条件が成立した時（真：True）に実行される範囲です。このルールのおかげで、Pythonのコードは誰が書いても同じように整って見えるというメリットがあります。

2. else と elif：複雑な分岐

「そうでなければ（else）」や「別の条件として（elif）」を組み合わせることで、多方向への分岐が可能になります。

Python：多方向分岐 
score = 75

if score >= 90:
    print("ランク A")
elif score >= 70:
    print("ランク B")
elif score >= 50:
    print("ランク C")
else:
    print("残念、不合格です")

3. 論理演算：複数の条件をまとめる

Pythonの論理演算子は、記号ではなく `and` や `or` などの「単語」を使います。これにより、コードがまるで英文を読んでいるかのように直感的になります。

  • and: AもBも成立するとき（かつ）
  • or: AかBのどちらかが成立するとき（または）
  • not: 条件を反転させる（〜ではない）
Python：論理演算と範囲 
# Pythonならではの書き方！
if 10 <= score <= 100:
    print("有効なスコア範囲内です")

if is_user and not is_banned:
    print("ログインを許可します")

4. Pythonの新機能：match-case (Python 3.10+)

最新のPythonでは、多数の選択肢をよりスマートに捌く **`match-case`** が導入されました。他言語の switch 文に相当しますが、より高度な「パターンマッチング」が可能です。

Python：match-case文 
status = 404
match status:
    case 200:
        print("成功")
    case 404:
        print("ページが見つかりません")
    case _:
        print("不明なエラー") # _: は「その他すべて」

🚀 真偽判定の落とし穴：何が Falsy か？

Pythonでは、単なる数値やリストも条件式に入れられます。以下のものは「偽（False相当）」とみなされます。
・数値の `0`
・空のリスト `[]` や辞書 `{}`
・空の文字列 `""`
・`None`（何もないことを表す値）
これ以外は基本的に「真」として扱われます。

よくある質問 (Q&A)

Q: インデントはスペースとタブ、どちらが良いですか？

A: Pythonの公式ルール（PEP 8）では **「半角スペース4つ」** が強く推奨されています。混ぜるとエラーの原因になるので、エディタの設定でタブをスペースに変換するようにしておきましょう。

Q: 1行で if 文を書くことはできますか？

A: はい、条件演算子（三項演算子）が使えます。 `print("合格") if score >= 80 else print("不合格")` のように記述できますが、複雑な場合は可読性のために通常の if 文を使いましょう。

Q: pass 文とは何ですか？

A: 「何もしない」ことを表す命令です。if 文を書いたけれど中身が決まっていないとき、エラーを避けるためにとりあえず `pass` と書いておきます。

まとめ

条件分岐はプログラムに「知性」を与える第一歩です。インデントを正しく使いこなし、`and` や `or` でスマートに条件を記述できるようになれば、あなたはもう初心者卒業です。美しいインデントは、美しいプログラムの第一歩！
