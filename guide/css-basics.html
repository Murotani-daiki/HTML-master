CSSの基礎知識

ウェブサイトに美しさと使いやすさを加えるデザインの技術

CSSとは何か？：Webページに命を吹き込む「デザインの魔法」

CSS（Cascading Style Sheets）は、HTMLで作られた「骨組み」に対して、色や形、配置といった「装飾」を施すための言語です。HTMLが「何がそこにあるか」を定義するのに対し、CSSは「それがどのように見えるか」を決定します。

CSS（カスケード）という名前には「上から下へと流れる」という意味があり、複数の指定が重なった時に、どのルールを優先するかを決める強力なルールセットが備わっています。この仕組みを理解することが、思い通りのデザインを実現するための第一歩となります。

1. CSSの書き方：セレクタ・プロパティ・値

CSSは非常にシンプルな構造をしています。「どの要素（セレクタ）」の、「何（プロパティ）」を、「どう変える（値）」かというセットで記述します。

CSS：基本の構文 
/* セレクタ { プロパティ: 値; } */
h1 {
    color: #3b82f6;      /* 文字色をブルーに */
    font-size: 32px;      /* 文字サイズを32pxに */
    text-align: center;   /* 中央揃えに */
}

波カッコ `{ }` の中に複数の命令（宣言）を書くことができます。各命令の末尾には必ずセミコロン `;` を忘れないようにしましょう。これは、複数の命令を区切るための重要な記号です。

2. カスケード（優先順位）の仕組み

同じ要素に対して複数のスタイルが指定された場合、CSSは以下のルールに従ってどれを採用するかを決定します。

  • 後から書いたものが勝つ: 同じ強さの指定であれば、ファイル内でより下に書かれたルールが優先されます。
  • 具体性は力なり: h1 というタグ指定よりも、特定のID（例: #main-title）を指定した方が優先されます。
  • スタイルの継承: 親要素に指定した文字色やフォントは、特に指定がなければ子要素にも引き継がれます。

3. CSSを適用する3つの方法

ウェブページにCSSを読み込ませるには、主に3つの方法があります。開発現場では「外部ファイル」による管理が最も一般的です。

  • 外部CSS（推奨）: <link> タグを使って .css ファイルを読み込みます。複数のページでデザインを共有できるため、メンテナンス性が最高です。
  • 内部CSS: HTMLの <head> 内に <style> タグを書いて記述します。そのページ専用のスタイルを書きたい時に使います。
  • インラインCSS: HTMLタグに直接 style="..." と書きます。特定の一箇所だけを急ぎで変えたい時の最終手段です。
HTML：外部ファイルの読み込み例 
<head>
    <link rel="stylesheet" href="style.css">
</head>

4. ボックスモデルの概念

CSSを扱う上で、全ての要素は「四角い箱」であると考える必要があります。この箱（ボックス）の隙間をどう調整するかが、レイアウトの基本です。

  • Margin: ボックスの外側の余白（他の要素との距離）
  • Border: ボックスの枠線
  • Padding: 枠線と中身（文字など）の間の内側の余白
  • Content: 実際のテキストや画像が表示される領域

よくある質問 (Q&A)

Q: CSSが効きません！どうすればいいですか？

A: まずは「スペルミス」と「セミコロン ; の付け忘れ」を確認しましょう。また、ブラウザの「検証ツール（F12）」を使うと、どのスタイルが打ち消されているか一目で分かります。

Q: コメントはどうやって書きますか？

A: /* 隠したい内容 */ で囲みます。HTMLのコメントとは書き方が違うので注意してください。メモを残したり、一時的にスタイルを無効化したりするのに便利です。

Q: IDとクラス、どちらを使うべきですか？

A: 基本的には **クラス (.class-name)** を使いましょう。IDはページ内に一つしか存在できないため、デザインの使い回しがきかなくなります。クラスなら、複数の要素に同じデザインを簡単に適用できます。

まとめ

CSSは、あなたのウェブサイトを「ただのテキスト」から「感動を与えるデジタル体験」へと進化させる魔法のツールです。まずは色を変える、中央に寄せる、といった小さな一歩から始めてみましょう。コードを一行書くたびに、画面がプロフェッショナルな表情に変わっていく楽しさを、ぜひ味わってください！
