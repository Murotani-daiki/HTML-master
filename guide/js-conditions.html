条件分岐

if文を使ってプログラムに「判断」をさせる

「もし〜なら」を組み立てる

私たちの日常生活と同じように、プログラムも「状況に応じて行動を変える」必要があります。例えば、「スコアが80点以上なら合格」、「ボタンが押されたならメッセージを表示」といったロジックを担うのが条件分岐です。これができると、プログラムは単なる計算機から「考える機械」へと進化します。

基本の if 文：条件による実行

JavaScriptの条件分岐の基本は `if` 文です。`if` の後ろの丸括弧 `()` の中に条件を書き、それが「正しい（true）」場合にのみ、波括弧 `{}` の中の処理が実行されます。

JavaScript 
const score = 85;
                        
if (score >= 80) {
    // 条件が満たされたときだけ実行される
    console.log("合格です！");
}

else と else if：分岐を増やす

「そうでなければ」を表す `else` や、「そうでなくて、もし〜なら」という追加の条件を表す `else if` を使うことで、複雑な分岐を作ることができます。

JavaScript 
const time = 15; // 15時

if (time < 12) {
    console.log("おはようございます");
} else if (time < 18) {
    console.log("こんにちは");
} else {
    console.log("こんばんは");
}

比較演算子の落とし穴：=== と ==

JavaScriptには「等しい」を表す演算子が2つあります。ここが初心者にとって最初の壁ですが、モダンな開発では答えはシンプルです。

💡 必ず `===` (厳密等価) を使おう

  • `===` (厳密等価): 値だけでなく「型」も同じかどうかチェックします。
    例: `5 === "5"` は false（数値と文字だから）
  • `==` (等価): 型が違っても、無理やり変換して比較します。
    例: `5 == "5"` は true（文字の"5"を数値の5とみなす）

バグの原因になるため、基本的に `==` は使わず、常に `===` を使う癖をつけましょう。

論理演算子：かつ、または

複数の条件を組み合わせたい時は、論理演算子を使います。

  • && (AND): A かつ B
  • || (OR): A または B
  • ! (NOT): 〜でない（否定）
const isMember = true;
const hasCoupon = false;

// 会員 または クーポン持ち なら割引
if (isMember || hasCoupon) {
    console.log("割引適用！");
}

Truthy と Falsy：真偽値のようなもの

JavaScriptでは、`true/false` 以外の値も条件式に使えます。以下の値はすべて「偽（false）」として扱われます（Falsyな値）。

  • `false`
  • `0` (ゼロ)
  • `""` (空文字)
  • `null`
  • `undefined`
  • `NaN` (数値ではない)

これら以外はすべて「真（true）」として扱われます（Truthyな値）。これを利用して、「データがあるかどうか」を短く書くテクニックがよく使われます。

Switch文：たくさんの分岐

特定の変数の値によって処理を細かく分けたい場合、`if...else if` を連ねるよりも `switch` 文を使ったほうがスッキリ書けることがあります。

const fruit = "apple";

switch (fruit) {
    case "apple":
        console.log("リンゴです");
        break; // 忘れずに！
    case "orange":
        console.log("ミカンです");
        break;
    default:
        console.log("その他の果物です");
}

まとめ

条件分岐はプログラミングの「思考力」そのものです。複雑な条件を組み立てられるようになれば、ゲームの判定やユーザー認証など、あらゆる高度な機能を作れるようになります。まずは基本の `if` と `===` をしっかりマスターしましょう。
