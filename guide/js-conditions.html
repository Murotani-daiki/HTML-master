条件分岐

if文を使ってプログラムに「判断」をさせる

「もし〜なら」を組み立てる：思考するプログラム

私たちの日常生活は選択の連続です。「信号が赤なら止まる」「雨が降っているなら傘を持つ」といった判断を、プログラムの世界で実現するのが **条件分岐** です。

適切な条件分岐を記述できるようになると、プログラムは単なる命令の羅列から、状況を自ら判断して最適に動く「知的なシステム」へと進化します。Webアプリケーションにおけるログイン判定、ゲームの当たり判定、入力フォームのバリデーションなど、あらゆる動的な処理の基盤となります。

基本の if 文とスコープ

最も基本的な構文です。カッコ `()` 内の条件式が評価され、その結果が **true（真）** であれば、続く波カッコ `{ }` 内のコード（ブロック）が実行されます。

JavaScript 
const score = 85;
if (score >= 80) {
    console.log("合格です！"); // scoreが80以上なので実行される
}

else と else if：複雑な分岐の構築

条件が満たされなかった場合の「さもなくば (else)」や、別の条件をさらに追加する「さもなくば、もし (else if)」を組み合わせることで、多段階の判断ロジックを作ることができます。

JavaScript：多段階分岐 
if (score >= 90) {
    console.log("素晴らしい！Sランクです");
} else if (score >= 70) {
    console.log("合格！よく頑張りました");
} else {
    console.log("不合格... 次回に期待しています");
}

✅ 比較演算子の黄金律：=== を使おう

JavaScriptには `==`（等価）と `===`（厳密等価）がありますが、原則として **常に `===` を使うべき** です。

  • `1 == "1"` は **true** になります（勝手に型を変換してしまいます）。
  • `1 === "1"` は **false** になります（型まで正しくチェックします）。

意図しない挙動（バグ）の多くは `==` による暗黙の型変換から発生します。厳密なチェックを心がけましょう。

論理演算子：複数の条件を束ねる

「AかつB（AND）」や「AまたはB（OR）」といった複雑な条件は、論理演算子を使って表現します。

  • && (AND): すべての条件が true の時だけ全体が true になります。
  • || (OR): いずれか一つの条件が true なら全体が true になります。
  • ! (NOT): 条件を反転させます（trueならfalse、falseならtrue）。

ショートサーキット評価（短絡評価）の知恵

JavaScriptの論理演算子は、結果が確定した時点で評価を打ち切ります。例えば `A || B` でAが true なら、Bが何であれ全体は true なので、Bの処理は実行されません。これを利用して、エラーを防ぐ高度な書き方がされることもあります。

真偽値の正体：Truthy と Falsy

JavaScriptでは、Boolean型（true/false）そのものでなくても、if文の条件式に入れると自動的に true か false か判断される仕組みがあります。

以下の値は **Falsy（偽とみなされる値）** と呼ばれ、それ以外はすべて **Truthy（真とみなされる値）** です。

  • `false`
  • `0`（数値のゼロ）
  • `""`（空の文字列）
  • `null`
  • `undefined`
  • `NaN`（計算不能な数値）

三項演算子：簡潔な分岐

単純な二者択一であれば、`if/else` を使わず一行で書くことができます。

JavaScript：三項演算子 
const status = score >= 80 ? "合格" : "不合格";
console.log(status);

よくある質問 (Q&A)

Q: if文が多すぎてコードが読みづらくなりました。

A: 分岐が多すぎる場合は `switch` 文の検討、あるいは「ガード句（条件を満たさない場合に早めに return して処理を抜ける手法）」を活用すると、ネスト（階層）が浅くなり読みやすくなります。

Q: empty array `[]` は true ですか false ですか？

A: **true** (Truthy) です！ 配列やオブジェクトは、中身が空であっても「存在している」とみなされます。中身があるかチェックしたい場合は `array.length > 0` のように記述が必要です。

Q: `&&` と `||` はどっちが優先されますか？

A: `&&` の方が優先順位が高いです。複雑な式を書く場合は `( )` で囲って意図を明確にしましょう。

まとめ

条件分岐はプログラミングの「思考力」そのものです。単なる文法の丸暗記ではなく、「どのような状況を想定し、どのように判断させるか（アルゴリズム）」を設計する楽しさを味わいましょう。正確な比較と論理的な組み立てができれば、あなたの作るアプリケーションはより盤石なものになります。
