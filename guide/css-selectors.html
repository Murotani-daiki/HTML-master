セレクタと詳細度

「どの要素」にスタイルを当てるか、その優先順位のルールを学ぼう

セレクタとは何か？：デザインを「狙い撃つ」ための技術

CSSの記述は「セレクタ」から始まります。セレクタとは、HTMLの中にある膨大な要素の中から、スタイルを適用したいターゲットを正確に見つけ出すための「検索条件」のようなものです。

単純にタグを選ぶだけでなく、特定の状態（マウスが乗っている時など）や、特定の順番にある要素だけを狙い撃つことができるようになると、CSSの表現力は飛躍的に向上します。このガイドでは、基本のセレクタから、優先順位を決定する「詳細度」の計算まで、プロが日常的に使う知識を深掘りします。

1. 基本セレクタの使い分け

まずは基本となる4つの手法を整理しましょう。

  • 要素セレクタ (`h1`, `p`): HTMLタグそのものを指定します。共通のベースデザインを作るのに適しています。
  • クラスセレクタ (`.card`): 汎用性が最も高い指定方法です。ページ内の複数の場所に同じデザインを適用したい時に使います。
  • IDセレクタ (`#header`): ページ内で唯一の要素を指定します。優先度が非常に高いため、多用するとスタイルの打ち消しが難しくなる点に注意が必要です。
  • 全称セレクタ (`*`): 全ての要素に一括で適用します。初期設定のリセットによく使われます。

2. セレクタの組み合わせ（結合子）

要素同士の関係性（入れ子構造など）を使ってターゲットを絞り込めます。

CSS：結合子の例 
/* 子孫結合子：.nav の中にある a 全て */
.nav a { color: blue; }

/* 子結合子：.list の直下にある li だけ */
.list > li { border-bottom: 1px solid; }

/* 隣接兄弟結合子：h2 の直後に来る p だけ */
h2 + p { margin-top: 0; }

3. 擬似クラスと擬似要素：状態と場所を指定する

HTMLには書かれていない「状態」や「パーツ」をデザインできます。

  • :hover: マウスが乗った時だけ変化させます（ボタンなどで必須）。
  • :nth-child(n): 「3番目の要素だけ」「偶数の要素だけ」といった指定が可能です。表（テーブル）の縞々模様などに便利です。
  • ::before / ::after: 要素の前後にある「空のスペース」を擬似的に作り出します。アイコンの表示や装飾的なラインを入れるのに重宝します。

4. 詳細度（Specificity）：スコアで決まる優先順位

同じ要素に別のスタイルが指定された場合、ブラウザは以下の「スコア制」で勝敗を決めます。

種類 スコア（目安） 例
インラインスタイル 1000点 style="..."
IDセレクタ 100点 #title
クラス / 擬似クラス 10点 .btn, :hover
要素 / 擬似要素 1点 div, ::before

!important という魔法の言葉を値の末尾に付けると、この計算を全て無視して最優先されますが、メンテナンス性を著しく下げるため、プロの現場では「最終手段」とされています。

よくある質問 (Q&A)

Q: クラス名を2つ重ねて指定できますか？

A: はい。`.btn.is-active` のようにスペースを入れずに繋げると、**「両方のクラスを同時に持っている要素」** にだけ適用されます。これにより詳細度も上がり、より強力な指定になります。

Q: 擬似クラスの `:` が1つ、擬似要素の `::` が2つなのはなぜ？

A: 歴史的な理由ですが、最新のルールでは「状態」を表すのがコロン1つの擬似クラス、「要素の一部」を作るのがコロン2つの擬似要素と区別されています。ただし、古いブラウザ対応のために擬似要素をコロン1つで書く習慣も残っています。

Q: 思い通りのスタイルが反映されません。

A: ほとんどの原因は「詳細度負け」です。ブラウザの検証ツール（Elementsタブ）を見て、自分の書いたスタイルが打ち消し線（横線）で消されていないか確認してみましょう。

まとめ

セレクタをマスターすることは、CSSという武器を自由自在に操れるようになることと同じです。詳細度のルールを味方につけ、効率的で美しいコードを目指しましょう！
